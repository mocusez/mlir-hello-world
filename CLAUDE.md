# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build & Run

Requires [pixi](https://pixi.sh) for environment management (provides MLIR 22, CMake, Ninja).

```bash
pixi run build          # configure + build all targets
pixi run helloworld     # build & run JIT executable
pixi run run-opt-mlir   # hello-opt → MLIR LLVM dialect output
pixi run run-opt-llvm   # hello-opt → LLVM IR output
pixi run clean          # remove build directory
```

Single target builds: `pixi run build-jit` (helloworld) or `pixi run build-opt` (hello-opt).

Dict test: `cd test/dict && pixi run -- ./run.sh` (needs clang in PATH or `CC` set).

## Architecture

This is an out-of-tree MLIR dialect project with two executables:

- **hello-jit** (`helloworld`) — programmatically builds MLIR IR with `hello.world` op and JIT-executes it
- **hello-opt** — reads `.mlir` files, applies lowering passes, emits MLIR LLVM dialect (`-emit=mlir-llvm`), LLVM IR (`-emit=llvm`), or JIT-executes (default)

### Dialect pipeline

```
TableGen (include/Hello/*.td)
  → generated C++ (*.h.inc, *.cpp.inc)
  → MLIRHello library (lib/Hello/)
  → linked into helloworld + hello-opt
```

### Hello dialect operations

- `hello.world` — lowers to printf("Hello, World!\n")
- `hello.dict.{create,put,get,delete,free}` — dictionary ops using `!hello.dict<K, V>` type, lowered to C runtime calls

### Lowering pass

`lib/Hello/LowerToLLVM.cpp` contains the single pass (`createLowerToLLVMPass()`) that converts all hello dialect ops to LLVM dialect. It chains conversions for Func, Arith, ControlFlow, MemRef, and SCF dialects. Dict ops lower to calls into an external C library (`dictlib.ll`).

### Key patterns

- Ops defined via TableGen in `include/Hello/HelloOps.td`, code-generated by `add_mlir_dialect(HelloOps hello)` in `include/Hello/CMakeLists.txt`
- DictType uses MLIR's `TypeStorageAllocator` pattern (storage in `HelloDialect.cpp`)
- All targets link against shared `MLIR`, `LLVM`, and `MLIRExecutionEngineShared` — do not mix with static MLIR component libraries (causes TypeID duplication crashes)
- Use `OpTy::create(builder, loc, ...)` API, not the deprecated `builder.create<OpTy>(loc, ...)`
